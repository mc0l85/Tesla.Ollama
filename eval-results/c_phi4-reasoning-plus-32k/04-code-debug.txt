The bug is that the function only checks for lists. In the example, the tuple (2, 3) is not a list, so it isn’t flattened; instead, it remains as a tuple in the output.

A corrected version that flattens both lists and tuples is:

-------------------------------------------------
def flatten(nested_list):
    result = []
    for item in nested_list:
        # Check if the item is either a list or a tuple
        if isinstance(item, (list, tuple)):
            result.extend(flatten(item))
        else:
            result.append(item)
    return result

# Test the function
print(flatten([1, (2, 3), [4, [5]]]))  # Now prints: [1, 2, 3, 4, 5]
-------------------------------------------------

Explanation:
• Originally, the function only checked if an item was a list using "if type(item) == list". This means that any other iterable (like a tuple) will not be flattened.
• In the corrected version, we use "if isinstance(item, (list, tuple))" so that both lists and tuples are recognized and recursively flattened.
• This change ensures that the tuple (2, 3) is flattened to 2 and 3, giving the expected output [1, 2, 3, 4, 5].
