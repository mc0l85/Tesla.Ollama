To solve this problem, we need to merge two sorted lists of integers into a single sorted list without using any built-in sorting functions. The solution should efficiently combine the elements from both lists while maintaining the sorted order.

### Approach
1. **Problem Analysis**: The problem requires merging two sorted lists into one sorted list. The key insight here is that since both input lists are already sorted, we can efficiently merge them by comparing elements from the start of each list and appending the smaller element to the result list. This approach ensures that the merged list remains sorted without needing to sort the combined list from scratch.
2. **Algorithm Selection**: We use a two-pointer technique where one pointer starts at the beginning of each list. We compare the elements at these pointers, append the smaller element to the result list, and move the corresponding pointer forward. This process continues until all elements from both lists are processed.
3. **Complexity Analysis**: The algorithm runs in O(n + m) time, where n and m are the lengths of the two input lists. This is optimal because each element is processed exactly once. The space complexity is O(n + m) to store the merged result.

### Solution Code
```python
def merge_sorted(list1, list2):
    """
    Merge two sorted lists of integers into a single sorted list.
    
    Args:
        list1 (list): A sorted list of integers.
        list2 (list): A sorted list of integers.
        
    Returns:
        list: A new sorted list containing all elements from list1 and list2.
    """
    i = 0
    j = 0
    merged = []
    
    while i < len(list1) and j < len(list2):
        if list1[i] <= list2[j]:
            merged.append(list1[i])
            i += 1
        else:
            merged.append(list2[j])
            j += 1
            
    while i < len(list1):
        merged.append(list1[i])
        i += 1
        
    while j < len(list2):
        merged.append(list2[j])
        j += 1
        
    return merged
```

### Test Cases

```python
# Test case 1: Both lists have elements
assert merge_sorted([1, 3, 5], [2, 4, 6]) == [1, 2, 3, 4, 5, 6]

# Test case 2: One list is empty
assert merge_sorted([], [1, 2, 3]) == [1, 2, 3]
```

### Explanation
1. **Initialization**: We initialize two pointers, `i` for `list1` and `j` for `list2`, starting at the beginning of each list. We also create an empty list `merged` to store the result.
2. **Merging Process**: 
   - While both pointers are within their respective list bounds, we compare the elements at the current pointers.
   - The smaller element is appended to `merged`, and the corresponding pointer is incremented.
3. **Remaining Elements**: After one list is exhausted, the remaining elements of the other list are appended to `merged` in order.
4. **Result**: The `merged` list, now containing all elements from both input lists in sorted order, is returned.

The test cases verify that the function correctly merges two non-empty sorted lists and handles the edge case where one list is empty. The assertions ensure the function behaves as expected under these conditions.
