To solve this problem, we need to merge two sorted lists of integers into a single sorted list without using any built-in sorting functions. The approach involves comparing elements from both lists sequentially and appending the smaller element to the result list until all elements from both lists are exhausted. This method efficiently combines the two sorted lists by leveraging their existing order.

### Approach
1. **Initialization**: Start with an empty list to store the merged result and two pointers initialized to the start of each input list.
2. **Comparison and Merging**: While both pointers are within their respective list bounds:
   - Compare the elements at the current pointers of both lists.
   - Append the smaller element to the result list and move the corresponding pointer forward.
3. **Append Remaining Elements**: Once one list is exhausted, append all remaining elements from the other list to the result list (since they are already sorted).

This approach ensures that the merged list maintains the sorted order by always selecting the smallest available element from the front of either list at each step.

### Solution Code
```python
def merge_sorted(list1, list2):
    """
    Merges two sorted lists into a single sorted list without using built-in sort functions.
    
    Args:
        list1 (list): A sorted list of integers.
        list2 (list): A sorted list of integers.
    
    Returns:
        list: A new sorted list containing all elements from list1 and list2.
    
    Example:
        >>> merge_sorted([1, 3, 5], [2, 4, 6])
        [1, 2, 3, 4, 5, 6]
    """
    result = []
    i, j = 0, 0
    while i < len(list1) and j < len(list2):
        if list1[i] <= list2[j]:
            result.append(list1[i])
            i += 1
        else:
            result.append(list2[j])
            j += 1
    while i < len(list1):
        result.append(list1[i])
        i += 1
    while j < len(list2):
        result.append(list2[j])
        j += 1
    return result

# Test cases
assert merge_sorted([1, 3, 5], [2, 4, 6]) == [1, 2, 3, 4, 5, 6]
assert merge_sorted([], [1, 2]) == [1, 2]
```

### Explanation
1. **Initialization**: The `result` list is initialized to store the merged elements. Pointers `i` and `j` start at the beginning of `list1` and `list2` respectively.
2. **Merging Loop**: The loop runs while both pointers are within bounds. The smaller of the current elements from `list1[i]` and `list2[j]` is appended to `result`, and the corresponding pointer is incremented.
3. **Appending Remaining Elements**: After one list is exhausted, the remaining elements from the other list are appended in order to `result` (since they are already sorted).
4. **Test Cases**: 
   - The first test case merges `[1, 3, 5]` and `[2, 4, 6]` into `[1, 2, 3, 4, 5, 6]`.
   - The second test case handles merging an empty list with `[1, 2]`, resulting in `[1, 2]`.

This approach efficiently merges two sorted lists in linear time, O(n + m), where n and m are the lengths of the input lists, ensuring optimal performance.
