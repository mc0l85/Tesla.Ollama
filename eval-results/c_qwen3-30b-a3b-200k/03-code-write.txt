To solve this problem, we need to merge two sorted lists of integers into a single sorted list without using any built-in sorting functions. The solution involves a two-pointer technique to efficiently combine the lists while maintaining the sorted order.

### Approach
1. **Initialization**: Start with an empty list to store the merged result.
2. **Two Pointers**: Use two pointers, `i` for the first list and `j` for the second list, both initialized to 0.
3. **Comparison and Merging**: 
   - While both pointers are within their respective list bounds, compare the elements at the current pointers.
   - Append the smaller element to the result list and increment the corresponding pointer.
4. **Append Remaining Elements**: After one list is exhausted, append all remaining elements from the other list to the result (since the remaining elements are already sorted).

This approach efficiently merges the two sorted lists in linear time, O(n + m), where n and m are the lengths of the two input lists.

### Solution Code
```python
def merge_sorted(list1, list2):
    """
    Merges two sorted lists into a single sorted list without using built-in sort functions.
    
    Args:
        list1: A sorted list of integers.
        list2: Another sorted list of integers.
    
    Returns:
        A single sorted list containing all elements from both input lists.
    
    Example:
        >>> merge_sorted([1, 3, 5], [2, 4, 6])
        [1, 2, 3, 4, 5, 6]
    """
    result = []
    i, j = 0, 0
    while i < len(list1) and j < len(list2):
        if list1[i] <= list2[j]:
            result.append(list1[i])
            i += 1
        else:
            result.append(list2[j])
            j += 1
    while i < len(list1):
        result.append(list1[i])
        i += 1
    while j < len(list2):
        result.append(list2[j])
        j += 1
    return result

# Test cases
assert merge_sorted([1, 3, 5], [2, 4, 6]) == [1, 2, 3, 4, 5, 6]
assert merge_sorted([], [1, 2, 3]) == [1, 2, 3]
```
